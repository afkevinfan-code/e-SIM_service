# DATA DICTIONARY: 上網卡銷售報價查詢

## Purpose
- 本文件記錄上網卡銷售報價查詢專案中的主要資料檔、欄位用途與報價注意事項。
- 產出報價、商品文案或修改 `quote_lookup.py` 時，應以 Excel 內的實際資料為準，不要自行推測價格、流量、天數、效期或支援國家。

## Source Files

### `card.xlsx`
- 用途：實體 SIM 卡商品資料。
- 目前工作表：`Sheet0`
- 目前資料規模：約 1,722 列、6 欄。
- 目前欄位：`商品名称`、`单价(元)`、`報價成本`、`使用地`、`商品描述`、`备注`

### `eSim.xlsx`
- 用途：eSIM 商品資料。
- 目前工作表：`Sheet0`
- 目前資料規模：約 4,421 列、11 欄。
- 目前欄位：`商品名称`、`单价(元)`、`報價成本`、`商品编码`、`商品动态`、`使用地`、`商品类型`、`商品描述`、`最晚使用日期`、`备注`、`激活方式`

## Field Definitions

| 欄位 | 出現檔案 | 用途 | 注意事項 |
|---|---|---|---|
| `商品名称` | `card.xlsx`, `eSim.xlsx` | 商品主要名稱，通常包含地區、流量、天數或方案類型。 | 查詢與商品文案最重要欄位之一；不要任意改寫方案名稱。 |
| `单价(元)` | `card.xlsx`, `eSim.xlsx` | 原始單價欄位。 | 目前程式會原樣顯示；幣別、換算規則需依資料來源確認。 |
| `報價成本` | `card.xlsx`, `eSim.xlsx` | 報價查詢使用的成本或估算基準。 | `quote_lookup.py` 目前用此欄估算或排序；產出客戶價格前應人工確認售價策略。 |
| `使用地` | `card.xlsx`, `eSim.xlsx` | 支援使用的國家、地區或電信商。 | 查詢目的地主要依據；多國方案需留意涵蓋範圍與限制。 |
| `商品描述` | `card.xlsx`, `eSim.xlsx` | 流量、天數、通話、熱點、限速等商品內容描述。 | 商品文案與客戶說明應以此欄為核心依據。 |
| `备注` | `card.xlsx`, `eSim.xlsx` | 補充限制、注意事項或供應商備註。 | 報價與文案不得忽略重要限制。 |
| `商品编码` | `eSim.xlsx` | eSIM 商品代碼。 | 適合內部查找、對單與客服確認。 |
| `商品动态` | `eSim.xlsx` | eSIM 動態資料欄。 | 目前看起來可能與成本或供應商資料相關；用途需再依來源確認。 |
| `商品类型` | `eSim.xlsx` | 商品類型，例如虛擬卡 eSIM。 | 可用來區分 eSIM 與其他商品型態。 |
| `最晚使用日期` | `eSim.xlsx` | eSIM 最晚使用或效期相關資訊。 | 若有日期，商品文或報價需提醒效期；空白時不要自行推估。 |
| `激活方式` | `eSim.xlsx` | eSIM 啟用或安裝方式。 | 操作教學、客服提醒與商品文案的重要欄位。 |

## Current Query Logic
- `quote_lookup.py` 會讀取實體卡與 eSIM 資料，並建立 `Product` 物件。
- 查詢比對會把繁簡中文部分字元正規化，例如 `國`/`国`、`臺`/`台`、`歐`/`欧` 等。
- 查詢關鍵字會比對商品名稱、使用地、商品描述、備註、啟用方式與來源類型。
- 目的地命中 `使用地` 欄位時分數較高，其次是 `商品名称`，再來是其他描述文字。
- 若查詢包含天數，例如 `日本 5天`，程式會嘗試從商品名稱或描述判斷固定天數。
- 若商品屬於每日計價或每日流量方案，並且查詢帶有天數，程式可能用 `報價成本 * 天數` 估算。

## Quoting Rules
- 客戶報價前，必須確認方案名稱、價格、天數、流量、支援國家、啟用方式與備註。
- 不要把 `報價成本` 直接視為最終客戶售價，除非已確認售價策略。
- 若資料欄位空白或文字不清楚，應標示「待確認」，不要補不存在的規格。
- 多國方案要清楚列出主要支援國家與限制，不要只寫「歐洲通用」或「全球可用」。
- eSIM 方案要提醒客戶確認手機是否支援 eSIM、是否需掃 QR Code、是否需要抵達後啟用。
- 實體卡方案要提醒插卡、開通時間、熱點限制、限速規則與是否支援通話/簡訊。

## Recommended Output Format

| 欄位 | 說明 |
|---|---|
| 商品類型 | 實體卡或 eSIM |
| 商品名稱 | 取自 `商品名称` |
| 適用國家/地區 | 取自 `使用地` |
| 天數/流量 | 依 `商品名称` 與 `商品描述` 整理 |
| 成本/售價 | 依報價策略確認後填寫 |
| 啟用方式 | eSIM 取自 `激活方式`，實體卡依備註或人工確認 |
| 注意事項 | 取自 `备注` 與商品描述中的限制 |
| 適合旅客 | 例如短天數自由行、商務出差、多國旅遊、親子旅遊 |
